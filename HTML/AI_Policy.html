MATH 7243 – AI (ChatGPT) Policy & Prompt Library

This page explains how you may use AI tools (e.g., ChatGPT) in MATH 7243 to support learning while maintaining academic integrity and deep understanding.

Purpose Allowed Uses Not Allowed AI Use Disclosure Verification Checklist Prompt Library Bottom Line

1) Purpose

AI tools are allowed in this course as learning assistants, not as replacements for your own thinking. Use ChatGPT like you use a textbook, office hours, or a debugging partner.

Goal: Understanding first, submission second.

2) Allowed Uses

You may use ChatGPT to:

  • Clarify concepts from lecture or notes (e.g., “Explain bias–variance tradeoff with a geometric example.”)
  • Rephrase definitions or theorems in simpler language
  • Generate small synthetic examples for intuition
  • Debug your own code (after you attempt first)
  • Compare methods conceptually (e.g., SVM vs neural networks)
  • Generate practice problems and quiz questions
  • Draft outlines for reports or project proposals (you rewrite and verify)
  • Help interpret error messages and training behavior
✅ Good use

“Explain why L2 regularization shrinks weights, and show the effect on a 2D toy dataset.”

✅ Good use

“Here is my training curve. Suggest 3 likely issues and how to test each.”

3) Not Allowed Uses

You may not use ChatGPT to:

  • Submit AI-generated solutions as your own work
  • Generate full homework solutions or derivations
  • Produce project code without understanding every step
  • Bypass required proofs or algorithmic reasoning
  • Fabricate experiments, data, citations, or results
  • Copy answers without verification
  • Use AI to replace required intellectual effort
⛔ Not OK

“Solve the entire homework and give me the final PDF-ready solutions.”

4) AI Use Disclosure (Required)

If you use ChatGPT for any assignment, include an AI Use Statement at the end of your submission. This is not a penalty—it's professional transparency. 

AI Use Statement (example):
I used ChatGPT to help explain the intuition behind logistic regression loss functions
and to debug a PyTorch shape mismatch error. I verified all mathematical derivations
independently and rewrote all explanations in my own words.

6) Prompt Library for MATH 7243

Use these templates. Modify them to fit your assignment and include context from lecture notes when helpful. Click “Copy” to paste a prompt.

A. Conceptual Understanding

Regularization intuition
Explain regularization using geometry and the bias–variance tradeoff. Use a 2D toy example.
Loss functions
Give an intuitive explanation of cross-entropy loss vs MSE for classification, and explain why cross-entropy is typically preferred.
Kernel methods
Explain kernel methods in plain language, then connect the explanation to the kernel trick mathematically.

B. Mathematical Insight

Logistic regression gradient
Derive the gradient of logistic regression with L2 regularization step-by-step. Explain each term and the dimensions.
Convexity
Explain why convexity matters in optimization for ML. Give examples of convex vs non-convex objectives and what that implies for convergence.

C. Code Debugging

Loss not decreasing
Here is my code and training log (paste). The loss is not decreasing. List 3 likely causes, how to test each, and specific fixes.
Error message
Explain this error message and fix the bug. Also explain how to prevent it in the future: (paste error).
Vectorization
Rewrite this NumPy/PyTorch loop using vectorization. Explain the speedup and verify results match the original. (paste code).

D. Model Comparison

SVM vs LR vs NN
Compare SVM, logistic regression, and a shallow neural network for a small dataset. Discuss interpretability, bias/variance, and typical failure modes.
k-NN intuition
When can k-NN outperform linear models? Provide a concrete synthetic example and explain the geometry.

E. Experiment Design

Imbalanced metrics
Suggest evaluation metrics and plots for an imbalanced classification task. Explain how each metric can be misleading.
Data leakage
Help me design a data split strategy to avoid leakage for this dataset/problem: (describe). Include a checklist of leakage risks.

F. Project Writing Support

Report outline
Help outline a machine learning project report: problem, data, method, experiments, results, limitations, and next steps.
Rewrite paragraph
Rewrite this paragraph to be more precise and mathematically correct, while keeping it readable. (paste paragraph).

G. Practice Problems

Regularization quiz
Generate 5 conceptual quiz questions on regularization (L1 vs L2), with answers and brief explanations.
SGD practice
Create a homework-style problem about SGD vs batch GD, including a small numeric example and solution sketch.

Optional: When using these prompts, include course context (your notation, your constraints, and what you already tried).

7) Bottom Line

You may use ChatGPT in MATH 7243 as a tutor, not as an author. Your submitted work must always represent your understanding, reasoning, and verification.

If in doubt: ask first, submit later.

Quick Summary

Use AI to learn, debug, and practice. Do not use AI to generate final solutions or bypass required reasoning. Disclose AI use when applicable.

Allowed vs Not Allowed

✅ Allowed Explanations, examples, debugging, practice questions, outlines.

⛔ Not allowed Full solutions, fabricated results, copy-paste submissions, bypassing proofs.
